**Biofeedback Project - AEKA Heart Symphony**
ECG Biofeedback System Using AD8232 Heart Monitor

Author/ Data Manager: Aislynn Macco
Institution: Virginia Tech - Wearable Bioinstrumentation
Contact: aismac26@vt.edu

**Overview**
The AEKA Heart Symphony project is an interactive biofeedback system that responds to a user's physiological signals. The goal of the project is to 
combine biofeedback from heart rate (BPM) with adaptive music playback. The system monitors a user's heart rate in real time and automatically adjusts the 
tempo of music to match the user's current BPM. This creates an interactive experience where the user's biological signals influence the music being played. 
The project demonstrates how physiological sensing, embedded systems, and signal processing can be used together to create personalized experiences.

**How the System Works**

The system uses an ECG (electrocardiogram) sensor to detect the electrical signals generated by the heart. As the heart pumps blood through the body, 
electrical activity occurs through depolarization and repolarization of cardiac cells. These electrical signals spread through the body and can be detected 
at the skin using electrodes. The ECG signal is captured using the AD8232 Heart Monitor, which measures the electrical activity of the heart through electrodes 
placed on the body.

The signal processing pipeline works as follows:
1. ECG signals are detected through the electrodes placed on the body.
2. The signal is read by the AD8232 ECG sensor and transmitted to an Arduino.
3. The ECG signal is filtered between 5–15 Hz to remove noise and isolate the relevant cardiac waveform.
4. Peak detection identifies R-peaks in the ECG waveform.
5. The RR interval is calculated between successive peaks.
6. The heart rate (BPM) is derived from the RR interval.
7. The calculated BPM is printed in the MATLAB program.
8. MATLAB plays a music track whose tempo matches the user’s BPM for approximately 10 seconds.

This creates a real-time feedback loop where the user’s heart rhythm influences the music tempo.

**Hardware Requirements**

The following hardware components are required: Arduino microcontroller, AD8232 Heart Monitor ECG sensor, Breadboard, Jumper wires, Three ECG electrodes, 
Computer adapter / USB connection for Arduino

Electrode Placement

Three electrodes are required to detect the ECG signal: Upper Left Chest, Upper Right Chest, Right Lower Abdomen (Ground electrode)

This configuration allows the sensor to detect the electrical signal of the heart across the torso.

**Software Requirements**

The project uses the following software:
Arduino IDE – for reading ECG signals from the AD8232 sensor
MATLAB – for signal processing and adaptive music playback
CSV files – containing tempo information for the available music tracks

**Installation**

1. Install the Arduino IDE on your computer.
2. Install MATLAB.
3. Connect the AD8232 Heart Monitor to the Arduino using jumper wires and a breadboard.
4. Connect the Arduino to the computer via USB.
5. Upload the provided MATLAB code to begin reading ECG data.
6. Ensure MATLAB scripts and music tempo CSV files are located in the working directory.

**How to Use the System**

1. Attach the three ECG electrodes to the body as previously indicated.
2. Connect the electrodes to the AD8232 Heart Monitor.
3. Run the MATLAB script to process the signal.

The system will detect ECG peaks, calculate the RR interval, and derive the user’s BPM. With this information, MATLAB will play a music track with a tempo 
matching the user’s heart rate. The selected music track will play for approximately 10 seconds, creating an interactive biofeedback experience. This system 
provides a fun and interactive method for observing physiological signals and understanding how heart activity can be translated into real-time feedback.

**Technologies Used**

MATLAB: Used for signal processing, BPM calculation, and adaptive music playback. MATLAB also reads tempo data from CSV files and selects 
appropriate tracks based on heart rate.

Arduino: Used to interface with the ECG hardware and stream ECG signal data to the computer.

AD8232 Heart Monitor: A compact ECG sensor used to measure the electrical activity of the heart through skin electrodes.

Breadboard and Jumper Wires: Used to prototype and connect the ECG sensor to the Arduino microcontroller.

CSV Files: Used to store tempo information and metadata for the music tracks used by MATLAB.

Computer Adapter / USB Interface: Used to connect the Arduino hardware to the computer running MATLAB.
